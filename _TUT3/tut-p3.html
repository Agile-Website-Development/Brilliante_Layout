Development on your local machine

This tutorial assumes you're capable of running a clean WordPress installation on your own local server machine or on your online web server. Codex is your friend

If you're a newbie in WP theme development, I highly recommend to visit the WordPress Codex, the official documentation. There's a ton of useful information there to get you moving with the basics and advance stuff, with great links to external resources and tutorials.

Even when you become an expert, you'll find yourself visiting and revisiting the Codex to refresh your memory or to find out about some troubleshooting.

So, what do you need to get your own WP theme up and running? Well, at least these 2 files:

  • style.css
  • index.php

These two guys here need to be in a folder with the name of your theme, in the WordPress theme root (wp-content/themes/).

The style.css file header will look like this: 
		/*
		Theme Name: Brillante Layout
		Theme URI: https://github.com/tatygrassini/Brilliante_Layout
		Description: WordPress theme version for the Brillante Layout.
		Author: Taty Grassini
		Author URI: tatygrassini.github.com
		Version: 1.0
		Tags: brillante, white, blue, two-column, fixed-width, right-sidebar
		*/

This CSS comment is the information you read on your WP dashboard, on appearance/themes/.

The index file could be empty if you will, but always remember to create a 300 x 225 pixels image and name it screenshot. I already did that in Photoshop, so you don't have to worry about it.

Now, if you go ahead and check in your themes section, you'll see our brand new theme, with the Name, Version, Author and Descriptions.

You could also add the following in your index.php file:

		
Hey you, <?php bloginfo('name'); ?> theme!

Then, activate your new theme in your dashboard and... ta-da!

Congrats, you've just activated your Brillante Layout theme.

As you can see, that little PHP snippet there <?php bloginfo('name'); ?> is a WordPress function for displaying the site "name" dinamically.

bloginfo is one of many WP template tags we'll be using for this theme creation.

As I mentioned before, we need to replace our static content with those dynamic template tags. The bloginfo(); accepts parameters as stylesheet_url or template_url you can embed right into your (old) HTML code.

We'll get back later to explain line 19 (conditional body tag) and line 34 (search form).

See how WP handles absolute URLs in our generated source code.

And see also the navigation function wp_nav_menu, copied from the Twenty Ten theme. This is how navigation systems are coded now on WP themes. You don't want to know all the extra lines of code or tricks needed before, just to show a Home link.

But, we need a fallback for this new "nav". Maybe someone still using an earlier version of WP somewhere would like to test our awesome theme. We'll fix this in the...

functions.php

	<?php

	// This theme uses wp_nav_menu() in one location.
	register_nav_menu( 'primary', __( 'Primary Menu', 'brilliante_layout' ) );


	// Get our wp_nav_menu() fallback, wp_page_menu(), to show a home link.

	function brilliante_layout_page_menu_args( $args ) {
		$args['show_home'] = true;
		return $args;
	}
	add_filter( 'wp_page_menu_args', 'brilliante_layout_page_menu_args' );

	?>

We'll be adding code frequently to this file. Think about functions as your theme wild card for that extra flavor, functionality and fixings.

css/style.css

As you see on the screenshot, our current page arrow is gone, not in the hover state though. We should go and revisit the style.css file. No, not the style on the theme root, the one in the CSS folder. Let's fix this with little changes in the CSS.

All we need to do is inspect our code (Firebug or Chrome dev tool will do) for the new WP CSS class and replace it accordingly. Simple.

Before: 
	.top-content li.selected,.top-content li a:hover{background:url(img/nav-arrow.gif) no-repeat scroll center bottom transparent;}
After: 
	.top-content li.current_page_item,.top-content li a:hover{background:url(img/nav-arrow.gif) no-repeat scroll center bottom transparent;}
